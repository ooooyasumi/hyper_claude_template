# Design — 苏米老师播客视频

## 视频规格

| 项目 | 值 |
|------|-----|
| 分辨率 | 1080 × 1920（竖屏 9:16） |
| 帧率 | 30fps |
| 格式 | H.264 + AAC |
| 安全区 | 底部 240px 不留任何内容 |

| 蓝色辅助 | `#748CAB` | 嘉宾字幕、副标题 |

## 字体

| 用途 | 字体栈 | 字重 |
|------|--------|------|
| 标题/字幕/卡片正文 | `'Songti SC', 'Noto Serif SC', serif` | 700 |
| 标签/小字/辅助 | `'PingFang SC', 'Noto Sans SC', sans-serif` | 300 |
| 数字/英文 | `'Playfair Display', 'Georgia', serif` | 700 |

**注意：** 只用系统字体，不依赖 Google Fonts（离线渲染会失败）。

## 字幕规范

| 项目 | 值 |
|------|-----|
| 位置 | `bottom: 480px`（画面四等分的第三格） |
| 左右边距 | `left: 120px; right: 120px` |
| 定位方式 | **`position: absolute`（严禁 flex 流式布局）** |
| 字号 | 42px |
| 字重 | 700 |
| 行高 | 1.6 |
| 苏米颜色 | `#C9A96E` |
| 嘉宾颜色 | `#748CAB` |
| 每条时长 | 2-4 秒 |
| 去水词 | 呢/啊/呃/嘛/嗯/吧/哦/呀/哇 + 对对对/没错没错/OK对 |

**字幕必须由 `build_captions.mjs` 脚本生成**，时间戳来自 transcript.json 的 `words[].start_time/end_time`（逐字级别，毫秒精度）。严禁手动估算时间。

## 视觉元素类型

### 内容卡片（上半部动效区）

| 类型 | CSS class | 用途 | 触发时机 |
|------|-----------|------|----------|
| 关键词卡片 | `.keyword-card` | 核心概念展示 | 提到关键术语时 |
| 要点列表 | `.bullet-list` + `.bullet-item` | 3 个以上并列要点 | 列举时逐条展开 |
| 数据大字报 | `.data-burst` | 数字/百分比 | 数据出现时弹性出场 |
| 对比双栏 | `.comparison-row` + `.comparison-col` | 前后对比 | 两种方案/时代对比 |
| 流程步骤 | `.process-row` + `.process-step` | 步骤/流程 | 操作步骤逐颗弹出 |
| 概念标签组 | `.concept-tags` + `.concept-tag` | 3 个关键词 | 相关概念同时出现 |
| 提问卡片 | `.question-card` | 设问/引出话题 | 提出关键问题时 |
| 强调文字 | `.emphasis-text` | 金句/结论 | 核心观点总结时 |

### 装饰层（持续运动）

| 类型 | CSS class | 技术 | 说明 |
|------|-----------|------|------|
| 脉冲环 | `.pulse-ring` | CSS `@keyframes pulse-ring` | 5 个不同位置，关键时间点触发 |
| 浮动几何 | `.geo-deco` | CSS `@keyframes float-slow/float-rev` | 5 个不同形状，持续浮动 |
| 装饰点 | `.geo-dot` | CSS `@keyframes dot-breathe` + GSAP 弹入 | 每组 2-3 个，点缀空白 |
| 环境光球 | `.ambient-orb` | GSAP yoyo 浮动 | 4 个不同大小和运动轨迹 |
| 页面边框 | `.page-border-*` | GSAP borderColor 呼吸 | 双层金色边框 |
| 噪点纹理 | `::before` 伪元素 | SVG feTurbulence | 全屏叠加，opacity 0.025 |
| 径向光晕 | `::after` 伪元素 | radial-gradient | 顶部中心暖色光晕 |

### 字幕装饰

| 类型 | 说明 |
|------|------|
| 说话人指示点 | 字幕下方 8px 圆点，苏米金色/嘉宾蓝色，随字幕切换变色 |

## 动效系统

### 缓动函数（GSAP）

| 名称 | 缓动 | 用途 |
|------|------|------|
| `cardIn()` | `back.out(1.5)` scale 0.88→1 | 关键词卡片、概念标签 |
| `slideUp()` | `expo.out` y 50→0 | 普通卡片入场 |
| `burstIn()` | `back.out(2.5)` scale 0.3→1 | 数据大字报 |
| `emIn()` | `expo.out` 分主副两段 | 强调文字 |
| `dotsPop()` | `back.out(2)` staggered 0.12s | 装饰点组弹入 |

### CSS Keyframes（持续装饰动画）

| 名称 | 效果 |
|------|------|
| `pulse-ring` | 脉冲环 expand+fade，2.5s，触发后单次播放 |
| `float-slow` | 12-18s 循环浮动+微旋转 |
| `float-rev` | 反向浮动，13-16s 循环 |
| `dot-breathe` | 3s 呼吸缩放 1→1.5 |
| `shimmer` | 高亮扫过，translateX |
| `geo-spin` | 慢速旋转 |
| `line-extend` | scaleX 0→1 延伸 |

### 动效密度

- **平均每 2-3 秒一个视觉变化**
- 最长不超过 8 秒无视觉变化
- 轮换使用不同动画类型，避免连续 3 次以上同类型动画
- 动画跟随内容关键信息点，不是机械定时触发

### 轮换序列（避免重复）

缩放入场 → 强调文字 → 左右对比 → 数据大字报 → 关键词卡 → 装饰点 → 要点列表 → 概念标签 → 提问卡 → 流程步骤 → (回到缩放入场)

## What NOT to Do

- 不要加进度条
- 不要用 flex 流式布局放字幕（字幕必须 position: absolute）
- 不要手动估算字幕时间（必须用 build_captions.mjs 脚本）
- 不要连续 3 次以上使用同一种动画类型
- 不要让画面超过 8 秒没有任何视觉变化
- 不要在底部 240px 安全区放任何内容
- 不要用 Google Fonts（离线渲染会失败）
- 不要在 GSAP 中动画 visibility（只动画 opacity/transform）
- 不要用 `repeat: -1`（必须计算有限重复次数）
- 不要在 async/setTimeout 中构建 GSAP 动画（必须同步构建）
